# Bain 案例竞赛 - 团队仓库

## 团队简介

我们是一支四人小队，参与 Bain 案例竞赛。本仓库用于整理所有项目资料、研究成果和最终交付物。

## 仓库结构

```
├── People/              # 队员信息与联系方式
├── Drafts/              # 草稿与工作文档
├── Finals/              # 最终提交材料
├── Preprocessings/      # 数据预处理脚本与处理结果
├── Reference/           # 参考资料（案例指南、研究文献）
├── Reports/             # 分析报告与演示材料
├── Scripts/             # 数据分析与建模代码
├── Templates/           # 文档模板
├── AVALIABILITY.csv     # 队员时间可用性表
├── TASKS.md             # 任务分配与进度追踪
├── WORKFLOW.md          # 项目工作流程
└── meeting_notes.md     # 会议记录
```

## 目录说明

### `People/`
队员信息，包括姓名、角色分工和联系方式。每位队员有对应的 `.md` 简历文件和学生证 PDF。

### `Drafts/`
工作过程中的草稿文档，包括初步分析、框架草稿和迭代版本。

### `Finals/`
最终提交材料，包括定稿的演示文稿、报告正文和附录。

### `Preprocessings/`
数据清洗、转换和特征工程的脚本，也包含已处理好的数据集。

### `Reference/`
参考资料，包括案例分析指南、研究文献和其他参考材料。

### `Reports/`
分析输出，包括市场研究、数据可视化和战略建议。

### `Scripts/`
分析代码，包括数据处理、统计建模和可视化脚本。

### `Templates/`
常用文档模板，方便快速创建标准化文档。目前包含案例模板 PDF。

### `AVALIABILITY.csv`
队员时间可用性表，记录各成员的空闲时间段。

### `TASKS.md`
任务分配与进度追踪，位于根目录，便于团队查看当前任务状态。

### `WORKFLOW.md`
项目工作流程，定义各阶段的任务和交付标准。

### `meeting_notes.md`
会议记录，位于根目录，用于记录讨论要点和决策。

## 协作规范

- **分支策略**：重大交付物使用功能分支开发
- **交叉审核**：所有最终材料移动到 `Finals/` 前需至少一名队友审核
- **文档记录**：代码和报告保持良好注释，便于团队查阅

## 快速上手

1. 克隆本仓库
2. 查看 `TASKS.md` 了解当前任务分配
3. 通过 `WORKFLOW.md` 了解项目阶段和流程
4. 通过 `meeting_notes.md` 记录和查阅会议要点
