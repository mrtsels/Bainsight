## Repository Structure

```
Bainsight/
├── People/              # Team member profiles (md + PDF resumes/cards)
├── Drafts/              # Working drafts, case files, slide versions
├── Finals/              # Final deliverables (PPT + report backups)
├── Preprocessings/      # Data processing scripts and datasets
├── Reference/           # Case guides, research materials
├── Reports/             # Analysis outputs and visualizations
├── Scripts/             # Data analysis and modeling code
├── Templates/           # Document templates (case PDFs)
├── AVALIABILITY.csv     # Team availability schedule (1=available)

├── WORKFLOW.md          # Project workflow and phases
```

## Workflow Phases

1. **Phase 0 (Pre-competition)**: Team coordination, role assignment
2. **Phase 1 (Case Analysis)**: Receive case, team discussion, framework building
3. **Phase 2 (Report Writing)**: Draft sections, cross-review, integrate
4. **Phase 3 (Submission)**: Format, submit, confirm receipt

## Task Status Conventions

In TASKS.md, use these status markers:
- `[ ]` Not started
- `[W]` Work in progress
- `[D]` Awaiting review
- `[✓]` Completed

## Bain Presentation Style

- **Primary color**: Red (#E60000) + black/white/gray
- **Design**: Flat, minimal decoration, data-driven
- **Titles**: Concise, conclusion-driven
- **Layout**: Generous whitespace, low info density per slide

### Presentation vs Report Mode

| Mode | Density | Whitespace | Focus |
|------|---------|------------|-------|
| Presentation (for presenting) | Sparse | Large | One insight per slide |
| Report (for reading) | Dense | Compact | Complete reasoning, data support |

## File Naming

- Date format: `YYYY-MM-DD`
- Version: `v1`, `v2`, `v3`
- Example: `2026-04-21_analysis_v1.md`

## Communication

- **Daily communication**: WeChat group
- **Document collaboration**: OneDrive sharedoc (PPT)
- **Code and text**: This repository
- **Meeting cadence**: Every 1-2 days

## Key Principles

1. **MECE Framework**: Use for problem decomposition
2. **Cross-review**: All finals must be reviewed by at least one teammate before moving to Finals/
3. **Backup**: Keep versions in Drafts/ before finalizing
4. **Documentation**: Keep code and reports well-commented

## Critical Rules

- Case PDF goes to `Drafts/case_original.pdf`
- Final PPT: OneDrive sharedoc + backup to `Finals/presentation_final.pptx`
- Slide drafts saved to `Drafts/slide_drafts/` with version history
- Data goes to `Preprocessings/raw/` → processed to `Preprocessings/processed/`

## Audience Considerations

- **Investors**: Data-driven, conclusions first, clear charts
- **Company executives**: Strategic focus, execution details, action-oriented
- **Case context**: Problem diagnosis, feasible solutions, rigorous logic
