# Case Report 结构与内容规划指南

> 本文档为 Case Report 提供系统性框架指导，涵盖叙事架构、分析框架与财务分析工具。

---

## 一、整体叙事架构：答案优先（Answer-First）

**核心理念**：将核心结论前置，让评审在最短时间内抓住重点。

| 步骤 | 内容 | 目的 |
|------|------|------|
| 1. 介绍 | 重新定义问题，重申公司核心价值 | 证明抓住核心痛点，建立分析立足点 |
| 2. 提出建议 | 明确、具体、可操作的核心解决方案 | 亮出王牌，为后续分析定调 |
| 3. 分析 | 展示分析过程与商业框架应用 | 证明建议基于事实与数据 |
| 4. 执行方案 | 具体时间表与责任部门 | 将创意转化为可落地蓝图 |
| 5. 财务分析 | 成本、收益、关键财务指标 | 验证方案财务可行性 |
| 6. 总结 | 重申问题、建议与价值，行动呼吁 | 强化印象，确保核心信息被记住 |

**关键原则**：每一部分都应服务于"让评审做出决定"这一终极目标。

---

## 二、各步骤内容规划详解

### 1. 介绍（Introduction）

**禁忌**：不要在 Introduction 中介绍团队，评审不关心参与者。

**必须包含**：
- **重新定义**个案问题：不要照抄题目，用自己的语言精准定位核心挑战
- **重申公司核心价值观**：建立分析立足点，证明理解企业本质
- 简短的背景概述（1-2句话），引出后续分析

**常见错误**：
- 复述题目原文（缺乏独立思考）
- 开头就列举团队成员
- 铺垫过长，迟迟不进入正题

**示例框架**：
> 本案的核心挑战并非表面的 XX 问题，而是更深层的 XX 困境。该公司（客户名称）的核心竞争力在于 XX，我们的建议将围绕如何**放大这一优势**展开。

---

### 2. 提出建议（Recommendation）

**这是整个报告最关键的步骤**。

**核心解决方案必须满足以下条件**：

| 要求 | 说明 |
|------|------|
| 清晰 | 一句话能说明白 |
| 具体 | 有明确的行动而非模糊方向 |
| 可操作 | 能转交给执行层落地 |
| 有阶段命名 | 便于后续引用（如"三阶段计划"） |
| 有明确时限 | 设定可衡量的目标 |

**必须避免**：
- "提升品牌影响力"（太模糊）
- "加强数字化转型"（方向而非行动）
- "拓展市场"（缺乏具体路径）

**示例结构**：
> 我们建议实施**「凤凰计划」三阶段战略**：
> - **第一阶段（0-6个月）**：渠道整合，建立线上线下 unified customer experience
> - **第二阶段（6-12个月）**：数据打通，实现精准营销闭环
> - **第三阶段（12-18个月）**：生态扩展，跨界合作打开新增量市场

---

### 3. 分析（Analysis）

**目的**：回答"为什么给出这个建议"。

**核心要求**：
- 运用商业分析框架支撑论点
- 展示结论基于**事实与数据**
- 每一步推理都有据可查

**结构建议**：
- 分析方法论先行
- 数据/事实作为证据
- 结论自然导出（而非强行跳转）

**常用框架**（详见第三节）：
- SWOT / TOWS 分析
- PESTEL 分析
- 波特五力分析

---

### 4. 执行方案（Implementation）

**核心原则**：不可执行的策略不是好策略。

**必须包含**：

| 要素 | 说明 |
|------|------|
| 具体时间表 | 短期/中期/长期各阶段关键里程碑 |
| 责任部门 | 明确哪个部门主导、哪些部门配合 |
| 跨部门协作机制 | 说明如何协调，避免各自为战 |
| 资源需求 | 人力、预算、技术等关键资源 |
| 风险预案 | 执行过程中可能遇到的障碍及应对 |

**评审视角**：执行方案是验证你是否具备"策略家+执行者"双重能力的核心环节。

---

### 5. 财务分析（Financial Analysis）

**目的**：将策略建议转化为董事会能理解的语言。

**核心要素**：

| 分析维度 | 具体内容 |
|----------|----------|
| 成本估算 | 实施所需的全部投入（人力、技术、营销等） |
| 收益预测 | 方案带来的增量收入或成本节省 |
| 关键指标 | ROI、回收期、预期销售增长、利润率变化 |
| 敏感性分析 | 关键假设变动对结果的影响 |

**终极判据**：净现值（NPV）
- NPV > 0：方案值得投资
- NPV < 0：应放弃或重新设计

**财务分析三张表**：

| 报表 | 角色 | 回答的核心问题 |
|------|------|---------------|
| 资产负债表 | 快照 | 公司体质如何？杠杆水平是否合理？ |
| 损益表 | 影片 | 这个方案能带来多少额外收入/利润？ |
| 现金流量表 | 追踪器 | 利润能否兑现为现金？回款周期多长？ |

**核心心法**：利润是虚的，现金是实的。

---

### 6. 总结（Summary）

**本质**：强化型行动呼吁（Call to Action）。

**必须重申三点**：

| 要素 | 具体内容 |
|------|----------|
| 核心问题定义 | 一句话重述我们解决了什么 |
| 独特建议内容 | 我们方案的核心差异点 |
| 财务价值 | 量化预期收益（NPV、ROI） |

**目的**：确保在评审离开会议室后，核心结论仍然清晰。

---

## 三、分析阶段三大基础框架

### 1. SWOT 分析（内外盘点）

**基础四象限**：

| 维度 | 性质 | 示例 |
|------|------|------|
| S 优势 (Strengths) | 内部可控 | 品牌知名度高、渠道覆盖广、技术专利 |
| W 劣势 (Weaknesses) | 内部可控 | 供应链效率低、人才储备不足 |
| O 机会 (Opportunities) | 外部不可控 | 政策利好、新兴市场崛起 |
| T 威胁 (Threats) | 外部不可控 | 竞争对手入局、替代品威胁 |

**进阶应用：TOWS 交叉矩阵**

| 策略类型 | 组合 | 思考方向 |
|----------|------|----------|
| SO 策略 | 优势 + 机会 | 如何利用优势抓住机会？ |
| ST 策略 | 优势 + 威胁 | 如何用优势应对威胁？ |
| WO 策略 | 劣势 + 机会 | 如何克服劣势把握机会？ |
| WT 策略 | 劣势 + 威胁 | 如何将劣势与威胁冲击最小化？ |

**使用时机**：
- 需要系统性梳理内外部因素时
- 需要从分析过渡到策略制定时
- SWOT 是起点，TOWS 是终点

---

### 2. PESTEL 分析（宏观扫描）

**六因素框架**：

| 因素 | 审视内容 | 典型问题 |
|------|----------|----------|
| P 政治 (Political) | 政策法规影响 | 监管趋势如何影响行业？ |
| E 经济 (Economic) | 宏观经济环境 | 通胀、汇率、消费力如何变化？ |
| S 社会文化 (Social) | 消费者偏好变化 | 人口结构、价值观、生活方式如何演变？ |
| T 科技 (Technological) | 颠覆性技术趋势 | 哪些技术可能重塑行业格局？ |
| E 生态环境 (Environmental) | 可持续发展要求 | ESG 压力如何影响经营？ |
| L 法律 (Legal) | 法律法规约束 | 合规要求如何演变？ |

**产出映射**：
- PESTEL 的 O（机会）和 T（威胁）直接输入 SWOT
- 目的：避免隧道视野，确保策略未来 3-5 年有效

**使用时机**：
- 市场进入分析
- 行业趋势判断
- 跨市场扩张评估

---

### 3. 波特五力分析（行业战场剖析）

**五力定义**：

| 五力 | 含义 | 核心问题 |
|------|------|----------|
| 供应商议价能力 | 上游掌控力 | 供应商是否比你强势？ |
| 购买者议价能力 | 下游谈判力 | 客户是否有替换你的能力？ |
| 新进入者威胁 | 行业壁垒高低 | 门槛够高吗？后来者容易追赶吗？ |
| 替代品威胁 | 被取代风险 | 是否有更低成本的替代方案？ |
| 现有竞争者强度 | 行业内竞争激烈程度 | 竞争格局是红海还是蓝海？ |

**核心判断**：五力强弱决定行业盈利潜力。

**策略应对方向**：

| 五力状态 | 建议策略 |
|----------|----------|
| 供应商强势 | 纵向一体化/供应商多元化 |
| 购买者强势 | 差异化/转换成本设计 |
| 新进入者威胁高 | 构建护城河/规模经济 |
| 替代品威胁大 | 产品创新/用户粘性建设 |
| 竞争激烈 | 差异化聚焦/成本领先 |

**使用时机**：
- 判断战场是否值得投入
- 制定竞争策略前
- 行业吸引力评估

## 四、财务分析核心工具

### 三大报表角色定位

| 报表 | 比喻 | 评估重点 |
|------|------|----------|
| 资产负债表 | 快照 | 某一时点公司财务状况，评估体质与杠杆 |
| 损益表 | 影片 | 一段时间内的经营成果，评估收入与成本 |
| 现金流量表 | 现金追踪 | 现金进出流向，验证利润是否真实 |

### 终极决策公式：净现值（NPV）

**基础概念**：金钱的时间价值
> 今天 1 元 > 明天 1 元（因为今天的钱可以投资生利）

**计算逻辑**：
```
NPV = Σ (未来现金流入 ÷ (1+折现率)^n) - 初始投入
```

**决策法则**：
- **NPV > 0**：方案值得投资
- **NPV < 0**：应放弃或重新设计
- **多方案比较**：选 NPV 最高者

### 投资回报率（ROI）

```
ROI = (收益 - 成本) ÷ 成本 × 100%
```

### 回收期（Payback Period）

```
回收期 = 初始投入 ÷ 年均净现金流
```

## 五、核心心法总结

| 心法 | 说明 |
|------|------|
| **答案前置** | 结构先行，结论先行，掌握简报主导权 |
| **框架交叉** | 分析框架是工具，交叉运用方见深度 |
| **执行导向** | 一切策略必须可执行、可量化 |
| **财务验证** | 每条建议都必须有财务依据支撑 |
| **评审视角** | 每一页都要回答"这对决策有什么帮助" |

**终极检验**：如果评审只能记住一页内容，那一页应该是什么？
