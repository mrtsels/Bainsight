# Issue Tree: Instant Retail Co. 三四线城市 EBIT 转正路径

## Framework Used

**Profit Bridge × 城市组合矩阵（修订版）**

核心假设：三四线城市盈利困境的根因是"城市资源配置 + 用户质量 + 品类结构 + 竞争约束"四维失衡，而非市场规模不足。

> **修订说明（基于 REVIEW 反馈）**：
> - 增加"维度 4：竞争约束"（原树缺失）
> - 将"履约成本"独立成枝（原散落于城市比较中）
> - 将"补贴效率"从维度 2 中明确拆出作为独立子枝
> - 城市维度与品类/用户维度的重叠**是有意为之**：城市是品类结构与用户质量的**载体**，城市选择本质上是资源配置决策；两者的重叠不影响 MECE，因为决策层次不同

## Tree Structure

```
如何在 3 年内实现核心城市 EBIT 转正？
│
├── 维度 1：城市选择与资源配置
│   ├── 止血优先投资组
│   │   ├── C3 泉州 — 已盈亏平衡，大超合作最成熟（23%）
│   │   ├── C7 惠州 — 体量大（120k 单/天），已接近盈利
│   │   └── C8 威海 — AOV 最高（65元），履约成本最低（9.0元）
│   │
│   ├── 优化精细化运营组（控制补贴，不追加大规模投入）
│   │   ├── C1 徐州 — 体量最大（110k），亏损最大（2,500万），精细化潜力大
│   │   ├── C2 南昌 — 中等体量（85k），关注补贴效率
│   │   └── C6 南宁 — 体量较大（95k），有改善空间
│   │
│   ├── 评估观察组（选择性收缩，12-18 个月验证）
│   │   ├── C9 桂林 — 新开城（2022），亏损严重（1,300万）
│   │   ├── C10 赣州 — AOV 最低（45元），高价值用户最少（15%）
│   │   ├── C4 洛阳 — 履约成本最高（10.8元）
│   │   └── C5 宜昌 — 体量偏小（45k），增速尚可（20%）
│   │
│   └── 履约成本压降（密度效应）
│       ├── 订单密度提升 — 徐州 110k 单/天理论上密度充足，
│       │   但 AOV 低（58元）导致单均配送成本摊不开
│       └── 配送路线优化 — 次要杠杆（C8 威海已达 9.0 元，验证可行）
│
├── 维度 2：用户质量提升
│   ├── 提升高价值用户占比（20% → 30%）
│   │   ├── 补贴精准化：从"补单量"转向"补留存"
│   │   │   ├── 识别高潜力用户（参考 C8 威海 26% 画像特征）
│   │   │   └── 补贴从普惠制 → 定向激励（需用户行为数据支撑 — 数据缺口）
│   │   └── 会员体系：复购激励 + 等级权益 + LTV 培养
│   │
│   └── 补贴效率提升（从维度 2 明确拆出）
│       ├── 诊断：80% 补贴流向非高价值用户
│       ├── 措施：低价值用户补贴阶梯递减，逐步降低
│       └── 约束：不能骤然停补贴（会导致订单流失 — 见维度 4）
│
├── 维度 3：品类结构优化
│   ├── 提升高毛利品类占比
│   │   ├── 大超合作深化（泉州 23% vs 赣州 12%）
│   │   │   ├── 提升大超占比 → 降低拣货成本 + 稳定供应链
│   │   │   └── 关键疑问：大超合作是时间积累的果（泉州 2018 年开城），
│   │   │       还是可快速复制的因？— **需在 Step 5 验证**
│   │   └── 到家快消（毛利率 40-55%）占比提升
│   │
│   └── 控制低毛利品类
│       ├── 生鲜（毛利率 15-25%）— 控制损耗是核心，不是减少订单
│       └── 餐饮（毛利率 30-40%）— 竞争激烈（美团外卖主导），
│           避免直接补贴战，转为提升客单质量
│
└── 维度 4：竞争约束（新增）
    ├── 美团外卖 — 餐饮品类直接竞争，决定补贴下限
    │   └── 餐饮占订单 20-28%，若停补贴用户立即流失
    ├── 京东到家 — 高客单用户被分流，品类重叠
    │   └── 威胁泉州/惠州等高 AOV 城市的用户留存
    └── 线下超市 — 占订单 60-70%，是真正的替代竞争对手
        └── 突破方式：即时配送差异化（30 分钟 vs 超市购物）
```

## MECE Check（修订版）

### 五大 EBIT 驱动因子覆盖

| 驱动因子 | 覆盖维度 | 处理方式 | 状态 |
|----------|----------|----------|------|
| AOV | 维度 3 | 品类结构优化 | ✅ 充分 |
| 订单量 | 维度 1 | 城市选择（密度效应） | ✅ 充分 |
| 高价值用户占比 | 维度 2 | 用户质量提升 | ✅ 充分 |
| 履约成本 | 维度 1 | 履约成本压降子枝 | ✅ 独立成枝（原缺失） |
| 补贴支出 | 维度 2 | 补贴效率子枝（从维度 2 拆出） | ✅ 独立成枝（原模糊） |
| **竞争约束** | 维度 4 | 竞争约束维度 | ✅ 新增 |

### Mutually Exclusive 分析

四个维度分别回答不同决策问题：

- **维度 1（城市）**：在哪竞争？资源往哪投？
- **维度 2（用户）**：吸引谁？如何留住所吸引的用户？
- **维度 3（品类）**：卖什么？用什么供给满足需求？
- **维度 4（竞争）**：竞争对手会如何反应？我们的约束是什么？

**重叠说明（有意为之）**：维度 1 的城市同时承载维度 2（用户）和维度 3（品类）的特征——这是因为"城市"本身就是资源分配的载体，而非分类维度的重叠。两者处于不同分析层次：维度 1 是**资源配置决策**，维度 2/3 是**运营效率改善**。

### Collectively Exhaustive 分析

EBIT = (AOV × 订单量 × 高价值用户占比) − (履约成本 + 补贴支出) − 固定成本

五大因子全部被覆盖（见上表），固定成本通过"收缩评估城市"间接覆盖（节省固定成本分摊）。
